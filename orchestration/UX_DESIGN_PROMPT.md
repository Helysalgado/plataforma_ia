Actúa como un UX/UI Lead especializado en plataformas colaborativas académicas y científicas.

Estamos diseñando una plataforma institucional para compartir, versionar y validar recursos de IA en bioinformática y genética.

El diseño debe:

- Transmitir profesionalismo académico.
- Invitar a colaborar.
- Fomentar reputación y reconocimiento.
- Ser moderno (estilo SaaS).
- Ser limpio, claro y visualmente atractivo.
- No ser sobrecargado.
- Priorizar claridad y comunidad.

--------------------------------------------------
OBJETIVO
--------------------------------------------------

Diseñar el flujo E2E prioritario del MVP con una experiencia atractiva que motive a publicar y reutilizar recursos.

--------------------------------------------------
FASE 1 — PROPUESTA DE IDENTIDAD VISUAL
--------------------------------------------------

Define:

1. Personalidad visual (3–5 adjetivos)
2. Estilo (dashboard SaaS moderno, con cards y métricas)
3. Paleta sugerida (colores primario, secundario, acento)
4. Tipografía recomendada
5. Estilo de botones y badges (Sandbox / Validated)
6. Cómo representar reputación y métricas visualmente

--------------------------------------------------
FASE 2 — FLUJO E2E PRIORITARIO
--------------------------------------------------

Propón un flujo principal atractivo que genere valor completo.
Ejemplo:

Registro → Login → Dashboard colaborativo → Publicar recurso → Confirmación → Visualización pública → Votación

Explica por qué este flujo fomenta colaboración.

--------------------------------------------------
FASE 3 — DISEÑO DE PANTALLAS (UX + EMOCIÓN)
--------------------------------------------------

Para cada pantalla del flujo:

1. Nombre
2. Objetivo emocional
3. Objetivo funcional
4. Layout (estructura visual jerárquica)
5. Componentes principales
6. Elementos que fomentan colaboración
7. Cómo se visualiza reputación
8. Estados UI (loading, empty, error, success)

--------------------------------------------------
FASE 4 — WIREFRAMES LOW-FIDELITY
--------------------------------------------------

Genera wireframes estructurados en texto:

Ejemplo:

[HEADER - Logo + Navegación]
[HERO - Mensaje inspirador]
[CARDS - Recursos destacados]
[CTA - Publicar nuevo recurso]
[FOOTER]

--------------------------------------------------
FASE 5 — ELEMENTOS DE COLABORACIÓN
--------------------------------------------------

Diseña:

- Sistema visual de votos
- Indicador de Validated
- Métricas de uso
- Perfil con reputación acumulada
- Micro-interacciones que inviten a compartir

--------------------------------------------------
FASE 6 — PREPARACIÓN PARA IMPLEMENTACIÓN
--------------------------------------------------

Indica:

- Componentes reutilizables necesarios
- Secciones que deben ser modulares
- Riesgos UX
- Decisiones pendientes

Salida estructurada como:

/docs/ux/VISUAL_IDENTITY.md
/docs/ux/WIREFRAMES.md
/docs/ux/COLLABORATION_SYSTEM.md
/docs/product/E2E_PRIORITY_FLOW.md

Termina con:

- Riesgos de diseño
- Recomendaciones estratégicas
